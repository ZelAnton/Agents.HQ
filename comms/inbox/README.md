# `comms/inbox/` — пер-репо вид «что ждёт меня» (опционально)

Здесь могут лежать сгенерированные файлы `<repo>.md` — сводка открытых тредов,
где `awaiting` содержит этот репо. Это **кэш-представление**, удобное агенту репо,
чтобы не сканировать все треды вручную.

> **Источник правды — поле `awaiting` в `threads/*/thread.md`, не эти файлы.**
> Если `inbox/<repo>.md` расходится с тредами — верь тредам.

Формат `<repo>.md` (пример):

```markdown
# Inbox: <repo>

Обновлено: YYYY-MM-DD

| Тред | Тема | Последнее | Kind |
|---|---|---|---|
| T-20260609-pk08-rollout | ProcessKit 0.8 rollout | #02 от ProcessKit-rs | change-request |
```

Поддерживать эти файлы вручную не обязательно — их будет генерировать агент-оркестратор.
