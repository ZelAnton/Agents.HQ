# `tasks/` — кросс-репо задачи и единая очередь

Здесь живут **спеки кросс-репо задач** (`TASK-####-*.md`) и **единый индекс ВСЕХ задач**
([`QUEUE.md`](QUEUE.md)) — и кросс-репо, и single-repo. Протокол — в [`../README.md`](../README.md).

## Где лежит спека задачи

- **cross-repo** (≥2 репо): файл `tasks/TASK-####-<slug>.md`.
- **single-repo**: файл `../projects/<repo>/tasks/TASK-####-<slug>.md`.
- **В обоих случаях** задача попадает строкой в `QUEUE.md` — это единственное место,
  где видно последовательность, приоритеты и зависимости всех задач сразу.

## Счётчик

`TASK-####` — единый глобальный монотонный счётчик. Новый номер = max в `QUEUE.md` + 1.

## Жизненный цикл

`queued` → `ready` (зависимости выполнены) → `in-progress` → (`blocked`) → `done` / `cancelled`.

- **`depends-on`** — only-after: задача не `ready`, пока все её зависимости не `done`.
- **`parallel-safe-with`** — явно безопасно параллельно (не трогают одно и то же).
- **Claim:** оркестратор, начиная задачу, ставит `status: in-progress` + `assigned-to: <agent>`
  → второй прогон её не возьмёт.
- Завершённое переносится в `_archive/`, строка в `QUEUE.md` помечается `done`.

## Как поставить задачу

1. Возьми следующий `TASK-####`.
2. Скопируй `_templates/task.md` в нужное место (cross → сюда, single → в `projects/<repo>/tasks/`).
3. Заполни `repos`, `depends-on`, `parallel-safe-with`, `priority`, `origin`.
4. Добавь строку в `QUEUE.md` и при необходимости отрази в секции «Волны».

Откуда берутся задачи: из принятых идей (`IDEA → promoted`), из тредов (`comms`), из решений
(`DEC`), из миграции (`MIGRATION.md`).
