# `knowledge/` — кросс-проектная база знаний

Знания, которые помогают агентам понимать пространство в целом и взаимодействовать,
не наступая друг другу на ноги. Протокол — в [`../README.md`](../README.md).

## Что где

- **[`dependency-graph.md`](dependency-graph.md)** — граф зависимостей между репо + порядок сборки.
  Определяет, какие кросс-репо изменения обязаны идти строго после каких.
- **[`ownership.md`](ownership.md)** — границы ответственности: что чей репо, чтобы агенты
  не лезли в чужую зону. Таблица + пары `-rs`/`.NET`.
- **`research/`** — внешний материал: конкурентный анализ, идеи из чужих репо (мигрировано из корня).
- **`howto/`** — переиспользуемые инструкции (например, настройка релизного токена), на которые
  ссылаются задачи и `HT`.

## Где «карточка» каждого репо

Карточка проекта (назначение, публичный интерфейс, границы, связи) — это
`../projects/<repo>/card.md`. Эта папка (`knowledge/`) держит **кросс-репо** срез
(граф, границы, research, howto), а не описания отдельных проектов.

Индекс карточек: см. `../projects/*/card.md` (по одной на репо; processkit-py/go — `no-repo-yet`).
