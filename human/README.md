# `human/` — человек в контуре

Сюда агенты кладут то, что **не могут сделать сами** и где нужен ты. Два вида:

- **`tasks/HT-####-*.md`** — **ручные действия**: настроить секреты, права, branch protection,
  внешние сервисы, ревью, оплата, что-то в GUI и т.п.
- **`decisions/DEC-####-*.md`** — **ключевые решения**: выбор, который должен сделать ты,
  прежде чем агент продолжит.

Начинай работу с **[`INBOX.md`](INBOX.md)** — это дашборд всего открытого.

---

## Как отвечать (КАНОН — агенты читают именно так)

Главный принцип: **ответ пишется в том же файле, что и вопрос/задача.** Файл — это и есть
вопрос, поэтому привязка «на что ты ответил» получается автоматически, без отдельного
хранилища ответов. `INBOX.md` — только индекс и нотификация; авторитет — файл `HT`/`DEC`.

### Ответ на решение (`DEC-####`)

Открой файл решения и заполни блок `answer:` во frontmatter, затем переключи `status`:

```yaml
answer:
  decision: B            # id выбранного варианта (A/B/…) или 'other'
  note: "если other или нужны нюансы — пиши сюда"
  by: anton
  date: 2026-06-09
status: answered         # было: open
```

Хочешь развёрнуто — допиши секцию `## Мой ответ` в теле. Агент следит за `status: answered`,
берёт `answer.decision` + `answer.note`, разблокирует задачи из `blocks[]`.

### Отметка ручной задачи (`HT-####`)

Открой файл задачи, заполни `## Результат` (ссылка на PR/коммит, «секрет CRATES_IO_TOKEN добавлен»,
и т.п.) и переключи статус:

```yaml
status: done             # done | wontdo | blocked
```

Агент увидит `done`, прочитает `## Результат`, разблокирует задачи из `blocks[]`.

---

## Приоритеты

`P0` — блокирует работу прямо сейчас; `P1` — нужно скоро; `P2` — можно позже.
Разбирай сверху вниз по приоритету в `INBOX.md`.

## Жизненный цикл

- `DEC`: `open` → `answered` / `deferred`.
- `HT`: `todo` → `doing` → `done` / `wontdo` / `blocked`.
- Закрытое (`answered`/`done`/`wontdo`) агент-оркестратор убирает из активного списка `INBOX.md`
  (файлы при желании переносятся в `_archive/`, но история и так в git).
